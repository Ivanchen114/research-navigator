# 三模式架構 — 實作 spec（v3）

> 給實作這套架構的人。這是《網頁定位鐵律》的「實作版」——鐵律講「東西該放哪一層」，這份講「程式上怎麼蓋」。
>
> **狀態：v3，可進入 W14 試作。** v3 修訂：DepthBlock 改「收合」非「隱藏」、ModeSwitch 加模式指引、第一屏加字數限制、內容三分→四分（加「刪除／移出」）、加總原則。

> **總原則：DepthBlock 不是垃圾桶，RecordDrawer 不是搬家箱，ModeSwitch 不是遮羞布。**
> 真正的目標是：上課時學生第一眼知道現在做什麼；自學時有一條補課路線；繳交前知道自己填了什麼、還缺什麼。

---

## 0. 要解決什麼

網站要同時支援三件事，但**不能擠同一個畫面**：
1. **上課即時操作** — 教室裡，老師在場，學生要的是「現在做什麼／填哪／完成按哪」。
2. **非同步自學補課** — 缺課／補課的學生，沒有老師，要完整說明、範例、常見錯誤、**一條補課路線**。
3. **歷程紀錄與繳交** — 學生隨時要能檢視「我這週填了什麼、要交什麼」。

現況：三件事的內容全擠在同一頁、同一層 inline，所以頁面很重。

---

## 1. 核心架構洞察：三模式不是三個互斥的桶

自學模式不是「另一份內容」，而是 **上課模式 ＋ 深度展開**：

```
上課模式 ＝ 核心（活動素材 ＋ 精簡指令 ＋ 提示 ＋ input 元件）；DepthBlock 收合、只見標題
自學模式 ＝ 核心 ＋ 深度展開（DepthBlock 攤開）＋ 補課路線
紀錄抽屜 ＝ 本週所有 input 的「總覽 ＋ 匯出」view（不是把 input 搬走）
```

含意：
- 「核心」永遠顯示，兩種模式共用——**不複製**。
- 「深度補充」用 `DepthBlock` 包；上課模式**預設收合但標題可見、可手動展開**（不是完全隱藏——學生卡住時要找得到），自學模式預設展開。
- input 元件**留在 Step 裡**；紀錄抽屜是「聚合 view」，不是「搬家目的地」。

## 2. 模式設計

```
[ 🚀 上課跟做 ]  [ 📖 自學補課 ]            [ 📁 我的紀錄 ]
   ↑ 兩個主模式（互斥、切換）                  ↑ 抽屜（固定存在，可開合）
```

- **預設進入「上課跟做」**。
- **ModeSwitch 下方放一行小字指引**（不要太大），告訴學生該點哪個：
  ```
  上課跟做：老師在場時使用。
  自學補課：請假、補課或想看完整說明時使用。
  ```
- **「我的紀錄」不是第三種模式**——是「產出檢視」，做成可開合的抽屜／側欄，兩主模式下都能開。
- 模式狀態存 `localStorage`（`w{N}-view-mode`）——切到自學模式、重整不會跳回。

## 3. 元件分解

採「最小可行」路線，不強制全站 LessonShell 重寫（見 §10 Phase 7）。先做四個：

| 元件 | 職責 | API（草案） |
|---|---|---|
| `ModeSwitch` | 頂部模式切換 UI ＋ 模式指引小字 ＋ 管 `mode` state | `<ModeSwitch mode={mode} onChange={setMode} />` |
| `DepthBlock` | 包「深度補充」。上課模式：收合、標題可見、可手動展開；自學模式：預設展開 | `<DepthBlock title="想知道為什麼？">{深度 JSX}</DepthBlock>` |
| `RecordDrawer` | 本週所有 input 的**總覽＋匯出** view（**方案 A**：input 仍留各 Step，這裡只聚合呈現 + ExportButton） | `<RecordDrawer week="W14" fields={...} />` |
| `ModeProvider` | context，讓 `DepthBlock` 讀 mode、不用一路傳 props | 包在週次頁最外層 |

**RecordDrawer 為何用方案 A**：上課最理想是「讀任務→看例子→馬上填」。input 全搬抽屜＝每次填都要開抽屜＝增加摩擦。第一版：input 留 Step，抽屜只做總覽與匯出。

**DepthBlock 為何用「收合」非「隱藏」**：完全隱藏＝學生卡住時搆不到深度。收合（標題可見、可點開）＝預設乾淨、深度永遠一鍵可達。也因此上課模式會看到一排 DepthBlock 標題 stub——所以 §6-2 的標題詞表更重要。

## 4. 內容四分法則（migration 時的判斷標準）

每個區塊問，分成**四類**（不是三類——少了第四類，DepthBlock 會變雜物間）：

| 類型 | 判準 | 處理 |
|---|---|---|
| **核心** | 活動素材／精簡指令／提示；學生自己一定要讀到 | 上課與自學都顯示，不包 DepthBlock |
| **深度** | 完整說明／範例詳解／為什麼重要／常見錯誤——老師在場會口頭講掉的 | 包 `DepthBlock`（上課收合、自學展開） |
| **紀錄** | input 元件（ThinkRecord / ThinkChoice / SignatureBlock / ExportButton） | 留在 Step，同時被 `RecordDrawer` 聚合 |
| **刪除／移出** | 不適合放在學生頁面的——過細的教學說明、只對老師有用的、或根本冗餘的 | 移到教師講稿／完整教案，或乾脆刪除 |

**灰色地帶判準**：
- 「老師在場會不會用嘴巴講掉？」會 → 深度。
- 「這是給學生看的，還是給老師看的？」給老師 → 刪除／移出。
- 「包進 DepthBlock 只是因為不知道放哪？」→ 那就是該「刪除／移出」，不是深度。

## 5. 自學三狀態 ＋ 補課路線

每週 data 標一個 `selfStudyStatus`：

| 狀態 | 意思 | 自學模式怎麼處理 |
|---|---|---|
| `full` | 可完整自學 | 完整步驟與練習，學生能等價完成 |
| `partial` | 可部分補課 | 替代任務 ＋ **明確標示**哪部分無法取代課堂互動 |
| `needs-live` | 必須現場互動 | 準備與補救清單，**不宣稱能完整補完** |

**自學模式 ≠ 只是「多展開一些說明」。** `partial` 與 `needs-live` 週的自學模式頂部，必須有結構化**補課路線**：

```
自學補課路線：
① 先看哪一段
② 做哪個任務
③ 補哪個紀錄
④ 交什麼
⑤ 哪個部分需要找同學／老師補做
```

W1-W8 ＋ W14 初判（動工時逐頁確認）：

| 週 | 狀態 | 理由 |
|---|---|---|
| W1 模仿遊戲 | `partial` | 模仿遊戲＋AI-RED 可自學；小組討論、現場人機協作示範需課堂 |
| W2 問題意識 | `full`（待確認） | 多為個人思考與寫作 |
| W3 題目決定 | `full`（待確認） | 個人診斷為主 |
| W4 方法地圖 | `full`（待確認） | 個人判斷為主 |
| W5 操作型定義 | `full`（待確認） | 個人練習為主 |
| W6 海報博覽會 | `needs-live` | 走讀、組隊合題本質需真人 |
| W7 文獻搜尋 | `partial` | 搜尋可自學；證物鑑識大賽是小組活動 |
| W8 文獻偵探社 | `partial` | 三明治改寫可自學；同儕複診需真人 |
| W14（試點） | `partial`（待讀檔確認） | 圖表＋圖說可自學；現場示範與同儕看圖需課堂 |

## 6. UI 防爆規格（防 scope creep 回流）

### 6-1. 第一屏規格 ＋ 字數限制
每週**第一屏只能放三件事，且每件限字數**：

```
今天做什麼：最多 1 句、30 字內
為什麼做：　最多 1 句、40 字內
今天交什麼：最多 2 項（必要時分小組／個人）
```

範例（W14）：
```
今天做什麼：把分析表變成一張圖表和一段圖說。
為什麼做：　圖表要幫資料說話，不能只漂亮或說太滿。
今天交什麼：小組＝圖表＋圖說；個人＝W14 歷程 docx
```

其餘一律往下、進 DepthBlock、進自學模式、或刪除／移出。

### 6-2. DepthBlock 標題詞表（固定，不自由發揮）
`DepthBlock` 的 `title` 只能從這組選——而且因為上課模式會看到這些**標題 stub**，詞表更要克制：

```
想知道為什麼？
看完整範例
常見錯誤
AI 使用提醒
延伸補充
```

**不准用「重要說明」「補充」「注意」**——一旦所有東西都叫「重要」，學生就全部忽略。（不用「老師補充」——自學模式裡沒有老師。）

## 7. 紙上 migration 方法示範（以 W1 為例）

> 試點是 **W14**，不是 W1。W14 的紙上 migration 要先讀 W14 全檔才能寫——那是 Phase 1 的產出。這裡用 W1 示範「**怎麼做一份紙上 migration**」。

逐區塊四分（節錄，示範判斷方式）：

| 現有區塊 | 類型 | 備註 |
|---|---|---|
| TOP BAR / HeroBlock / CourseArc / TaskCard | 核心 | 第一屏套 §6-1 規格 |
| 各 Step 的 `StepBriefing` | 核心 | 「現在做什麼」——上課模式主角 |
| Step2 7 份 SUSPECTS | 核心 | **活動素材**，兩模式都要看得到 |
| Step2 揭曉真相的長解說 | 深度（`DepthBlock` 標題「想知道為什麼？」） | 上課收合、自學展開 |
| Step3 人機分工兩欄卡 | 深度（「延伸補充」） | |

## 8. 遷移 SOP（每週照這個跑）

1. **讀全頁**，依 §4 把每個區塊標成**四類**：核心／深度／紀錄／刪除-移出。產出該週紙上 migration。
2. **處理「刪除／移出」**：該移的移到講稿／教案，該刪的刪——先清掉，DepthBlock 才不會變雜物間。
3. **包 `DepthBlock`**：深度區塊包起來（標題用 §6-2 詞表），核心不動。
4. **加 `RecordDrawer`**：聚合本週 input 做總覽＋匯出——input 留 Step、**dataKey 一字不改**。
5. **加殼**：`ModeProvider` ＋ `ModeSwitch`（含模式指引小字）。
6. **標自學狀態**：`selfStudyStatus`；`partial`/`needs-live` 週加 §5 補課路線。
7. **套第一屏規格**（§6-1，含字數）。
8. **build 驗證**（`dist-tmp` 繞 iCloud，見 AI鐵律 §9）＋ grep 確認 dataKey 0 斷鏈。
9. **三情境測試**（見 §9）。

## 9. 驗收：三情境測試（取代泛問「難不難」）

每頁 migration 完，用三方 AI 或真學生跑：

- **情境 A（上課）**：我在教室，老師叫我打開 W{N}。
- **情境 B（自學）**：我請假，要自己補 W{N}。
- **情境 C（紀錄）**：我下課前要確認自己填完沒、要交什麼。

每情境問：① 第一步做什麼？② 要交什麼？③ 卡在哪一句？④ 有沒有看到「現在不需要看」的東西？

| Phase | 做什麼 | 學生有感？ |

| **3** | W14 加 `RecordDrawer`（總覽＋匯出，input 留 Step）＋ 套第一屏規格 ＋ 處理「刪除／移出」 | 是 |
| **4** | W14 三情境測試 → 調整架構、鎖定模板 | — |
| **5** | 架構定案 → 先 **W13/W15 補完分析叢集**（W13→W14→W15 同一主題鏈、共用 ResearcherRedlines）→ 再 **W1-W8**（使用者原始目標）→ 最後 **W9/W10**（重章節週留最後）。一週一 commit、各自 build 驗證 | 是 |

（順序修訂 2026-05-15：原為 W14→W1-W8→W9/W10。授課老師決定先補 W13/W15——W13→W14→W15 是同一個分析叢集、共用紅線系統，做成連貫一塊比跳 W1 有道理。）

## 11. 風險與鐵律

- **dataKey 不斷鏈**：migration 不准改任何 dataKey。搬位置 OK、改名不行。
- **不重複內容源**：核心內容只存在一處。不要為「自學模式」複製一份。
- **DepthBlock 不是垃圾桶**：不知道放哪的東西 → 走「刪除／移出」，不是塞進 DepthBlock。
- **Phase 6 非選配**：inline 底座不清，scope creep 會回來。
- **Tailwind JIT**：新元件不准動態組 class，顏色用完整靜態字串或 `var(--*)` inline style。
- **archive 不動**：`_archive/` 下的舊頁不在範圍。
- **Phase 1/6 對學生無感**：別因為規劃或地基階段沒有學生端改善，就誤判「沒效果」。
- **自學敘事可能要補寫**：若某週深度不足以支撐自學，要補寫——那不是「抽」是「寫」，工時不同。
- **試點頁雙觸碰**：W14 試作期先 inline 包 DepthBlock，Phase 6 才抽常數——W14 會被碰兩次，這是「先驗證再投資」刻意付的代價。
