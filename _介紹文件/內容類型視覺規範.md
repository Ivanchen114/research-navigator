# 內容類型視覺規範

> 建立日期：2026-05-16
> 目標：學生進任何區塊，不讀文字就知道「我要做什麼動作」
> 原則：認知負荷最小化——視覺結構先說話，文字只補細節

---

## 核心三層系統

```
READ  → 吸收知識    → 白底、無互動元素
DO    → 主動行動    → 有輸入區 / 勾選區，底色可辨別
REF   → 按需查閱    → 預設收合 or 灰底等寬字體
```

學生的視覺記憶只需要記住這三種。

---

## 七種內容類型 → 三層對應

| 內容類型 | 層 | StepBriefing label | 主要元件 |
|---------|----|--------------------|---------|
| 理解型 | READ | 學 | 概念卡、✗/✓ 對比卡 |
| 記憶邊界型 | READ | 注意 | ResearcherRedlines warning |
| 判斷型 | DO | 做 | ChartChoiceChecker、ThinkChoice |
| 流程執行型 | DO | 做 | Timeline sub-steps + Checklist |
| 練習型 | DO | 練 | ThinkRecord with scaffold |
| 工具參考型 | REF | — | PromptBlock、DepthBlock |
| 紀錄型 | DO | 交出 | ThinkRecord open-ended、外部連結指引 |

---

## READ 層規格

**用於**：概念解說、原則說明、✗/✓ 對比示例

### 視覺規則
- 背景：白色或 `var(--paper-warm)`（米白）
- Border：無，或僅底部細線做區隔
- 字色：`var(--ink)`（深）
- **不出現任何輸入框、勾選框、按鈕**（除非是折疊展開）
- 層次靠字重：標題 `font-bold text-[14px]`、內文 `text-[12.5px]`、補充 `text-[11px] text-[var(--ink-light)]`

### ✗/✓ 對比卡
```jsx
// 固定格式：左欄錯誤（紅）、右欄正確（綠）
<div className="grid grid-cols-2 gap-2">
  <div className="p-3 rounded bg-[#FEF2F2] border border-[#FCA5A5]">
    <span className="text-[10px] font-mono text-[#DC2626]">✗ 錯誤</span>
    ...
  </div>
  <div className="p-3 rounded bg-[#F0FDF4] border border-[#86EFAC]">
    <span className="text-[10px] font-mono text-[#16A34A]">✓ 正確</span>
    ...
  </div>
</div>
```

### 記憶邊界型（規則卡）
- 沿用 ResearcherRedlines 的 warning 格式
- 左側 4px 色條 + stage accent 色
- `ids=` 過濾，只在該步驟相關的規則出現

## DO 層規格

**用於**：需要學生主動輸入、選擇、勾選的任何區塊

### 共同視覺信號
- 區塊開頭有**明確動詞提示**（「勾選」「填寫」「選一個」「記在…」）
- 有清楚的輸入目標：網頁內填寫 **or** 去外部工具（Google Slides / Classroom）
- 完成後有視覺收尾：Checklist ✓ 變色、ThinkRecord 顯示已儲存

### 子類型規格

#### 流程執行型 — Timeline sub-steps
```
適用時機：同一 Step 內有 3+ 個依序子步驟
```
- 左側垂直連線 + 圓形編號 badge
- 純人工子步驟：深石板 `#1E293B`
- AI 輔助子步驟：品牌藍 `var(--accent)`
- 驗收子步驟：琥珀 `#D97706`
- 每個 sub-step 有標題 + mono 小字說明（不重複 StepBriefing 的分工原則）

#### 練習型 — ThinkRecord with scaffold
```
適用時機：學生要在安全情境中試做，允許錯
```
- scaffold 欄位（灰底、提示字）與學生輸入區（白底）視覺分離
- 有範例時：先收折，學生寫完再展開對照

#### 判斷型 — 選擇 + 記錄決定
```
適用時機：學生要在選項中做有根據的選擇
```
- 提供可點擊的選項（ChartChoiceChecker 風格）
- 決定後：提示「記在小組 Google Slides」（inline 小字，不用獨立卡片）

#### 紀錄型 — 指向外部
```
適用時機：成果要留在 Classroom / Google Slides，不在網頁內
```
- 不用 ThinkRecord，改用箭頭指引卡
- 格式：`→ 把以下內容寫在 [小組 Google Slides]，完成後到 Classroom 按「繳交」`
- 視覺：藍色箭頭 + 連結樣式，一行即可，不用大卡片

## REF 層規格

**用於**：工具模板、格式速查、AI prompt、選填補充

### 核心原則
> REF 層預設不佔閱讀流，學生主動需要時才展開或複製

### PromptBlock（AI prompt）
- 等寬字體、灰底
- 右上角複製按鈕
- 無需解釋文字——prompt 本身就是工具

### DepthBlock（選填補充）
- 預設收合
- 標題語意明確（不用「延伸補充」）：「N 值為什麼重要？」「常見錯誤 & 修 prompt 對照」
- **禁止把必看觀念放進 DepthBlock**

### 格式速查
- 表格或清單格式，scannable
- 字級 11-12px，不需大段說明

---

## StepBriefing 與三層的關係

StepBriefing 是**預覽層**，告訴學生這個 Step 有哪些層：

```
學  → 這個 Step 有 READ 內容（概念要理解）
做  → 這個 Step 有 DO 內容（要主動操作）
練  → 這個 Step 有 DO/練習內容（要試做）
交出 → 這個 Step 有紀錄型 DO（要去外部繳交）
注意 → 這個 Step 有記憶邊界型 READ（紅線要記住）
```

每個 label 顏色已對應層的語意，學生看 StepBriefing 一眼就能預期這個 Step 的節奏。

---

## 禁止事項

| 禁止 | 原因 |
|------|------|
| 同一資訊重複兩次（StepBriefing + section header 各說一遍） | 增加閱讀量，無新資訊 |
| 重要觀念放進 DepthBlock | 學生可能跳過，造成理解缺口 |
| `done=` 寫「進 Step N」 | StepEngine 的下一步按鈕已處理 |
| 分工原則獨立卡片重複三次 | 說一次放對位置 |
| 記錄指引放在工具/checker 之前 | 學生還沒完成決定，記什麼？ |
| 數字同時出現在 StepBriefing 和 section header | 讀者眼睛走兩遍同樣的地圖 |

## 改版優先順序（依認知負荷影響大小）

1. **DO 層 → 紀錄型**：把所有「去 Slides 寫」的大卡片改成 inline 一行指引
2. **REF 層 → DepthBlock 清查**：確保只放可跳過的補充，不放必看內容
3. **READ 層 → 概念卡統一**：✗/✓ 對比卡統一格式
4. **DO 層 → 流程執行型**：有 3+ 子步驟的 Step 統一用 Timeline
